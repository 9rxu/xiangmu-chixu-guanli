# 创建新项目与 GitHub 仓库

## 执行顺序

1. 确认项目中文名称、详细构想、目标用户、核心场景和仓库可见性。信息不足时填写可确认部分，把其余标为“待确定”，不要虚构。
2. 检查 Git、可用 GitHub 工具和当前登录身份。
3. 查找当前目录所属的 Git 根目录。选择该根目录之外的父目录；未指定位置时，默认在现有仓库的父目录创建同级项目目录。
4. 生成仓库名并在当前账号下检查冲突。
5. 输出名称依据、本地绝对路径与最终可见性。用户已明确要求创建时可继续；涉及公开私人内容或覆盖现有资源时必须停止确认。
6. 创建全新的本地项目目录，解析绝对路径并确认它不位于任何现有项目仓库内部，然后初始化独立 Git 仓库和 `main` 分支。
7. 使用 `assets/` 模板创建标准文档，按技术栈生成 `.gitignore`。
8. 复制完整仓库级 Skill 到 `.codex/skills/xiangmu-chixu-guanli/`。
9. 检查敏感信息，运行项目连续性验证和项目自身验证。
10. 创建一个全新的 GitHub 仓库；不要复用当前仓库的远程地址，也不要把新项目作为现有仓库的子目录提交。
11. 在用户授权范围内创建首次提交并推送到这个新远程仓库。
12. 重新读取远程状态；如状态文档因远程地址变化而更新，再提交一次文档修正。

## 独立仓库边界

- “创建新项目”始终意味着新目录、新 `.git` 和新 GitHub 仓库。
- 禁止在当前仓库内部创建新项目目录，即使该目录被 `.gitignore` 忽略也不允许。
- 禁止复用当前仓库的 `origin`，禁止把新项目直接提交到当前仓库。
- 若建议目录已经存在，先确认其是否为空且不属于其他仓库；否则使用新的无冲突目录名。
- 创建后用 `git rev-parse --show-toplevel` 验证新项目根目录就是新目录本身。
- “接管已有项目”不适用本节；接管模式继续使用已有仓库，不创建第二个仓库。

## 拼音仓库名

根据中文项目名或核心用途生成完整、可读的汉语拼音：

- 只使用小写 ASCII 字母、数字和半角短横线。
- 词语之间使用 `-`；不使用中文、空格、下划线或声调符号。
- 不使用英文意译、拼音首字母、无意义随机字符或完整长句。
- 优先保留能辨认项目用途的核心词，一般控制为 2–6 个语义词组。
- 多音字或专有名词有实质歧义时，先展示拆分依据；无法可靠判断时向用户确认。

示例：

```text
个人日记系统 -> geren-riji-xitong
史莱姆吞噬进化游戏 -> shilaimu-tunshi-jinhua-youxi
智能工作流管理工具 -> zhineng-gongzuoliu-guanli-gongju
安卓任务管理软件 -> anzhuo-renwu-guanli-ruanjian
```

拒绝：`gerenrijixitong`、`grrjxt`、`personal-diary`、`new-project`、`demo`。

创建前输出：

```text
项目中文名称：
建议仓库名称：
拼音拆分依据：
是否存在同名仓库：
最终仓库名称：
本地项目绝对路径：
仓库可见性：
```

同名时依次增加有意义的平台、类型或用途拼音限定词；仍冲突再加简短数字，例如 `geren-riji-xitong-anzhuo-2`。绝不覆盖原仓库。

## GitHub 简介

使用中文准确说明：

- 项目是什么；
- 解决什么问题；
- 面向谁或什么场景；
- 当前已经具备的核心特点。

保持简洁、具体，不堆口号，不把计划功能写成已实现，也不使用“这是一个项目”“正在开发”等空泛文字。

## README 详细介绍

README 必须根据真实项目填写，不保留模板占位符，至少包含：

1. 中文项目名、拼音仓库名和维护状态。
2. 项目介绍与创建背景。
3. 近期、首个完整版本和长期目标。
4. 目标用户与具体使用场景。
5. 核心功能，并明确区分已完成、开发中、计划中。
6. 技术方案；未知内容标记“待确定”。
7. 真实的项目结构、安装运行和测试方法。
8. 当前进度、开发计划和已知问题。
9. 项目文档与接手说明。
10. 贡献与开发规范、许可证状态和维护状态。

没有确定许可证时写：

> 本项目暂未确定开源许可证。在许可证确定之前，不代表授权他人复制、修改或分发本项目。

命令尚未配置时明确写“尚未配置”，不能编造可执行命令。

## 远程创建安全

- 只有用户明确要求创建 GitHub 仓库时才创建远程资源。
- 未指定可见性时默认私有，并在结果中醒目标明。
- 推送前确认远程 URL、登录账号和目标仓库名。
- 推送后通过远程分支和文件列表验证真实结果。
- 任何步骤失败都要区分“本地已完成”和“远程未完成”。
